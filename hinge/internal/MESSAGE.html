Message

The Message module (src.hinge.network.Message) is what Clients (src.hinge.network.Client) use to handle data coming to and from the server (and vice versa).
A Message can contain the following pieces of data, a command, a route, a data, an hmac, an error, and a num.

  • command: what the server/client is supposed to do as a result of the Message.
  • route: a tuple containing the source client's ID and the destination client's ID, sometimes the source/destination can be 0 which is the server ID.
  • data: the data (usually encrypted) to be sent to the destination.
  • hmac: the hmac key to decrypt the data (the SMP (Socialist Millionaire Protocol) is in place to detect MITM attacks but not prevent them,
    • which is why there is a feature for nicks to authenticate themselves).
  • error: the error if the Message is invalid.
  • num: the specific number of the Message.
